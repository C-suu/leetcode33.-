### （1）题目白话文及生僻概念解释

**题目白话文解释**
给定一个原本按照从小到大排好序的数字列表。这个列表被从某个未知的切分点“切开”，并把前半部分整体平移到了列表的最末尾。这种操作被称为“旋转”。现在的核心任务是，在这个已经被打乱了整体顺序的列表中，寻找一个特定的数字 `target`。如果找到了，就返回它所在的位置编号（即索引下标）；如果找不到，就返回 `-1`。并且，查找过程的效率必须很高，时间复杂度要求达到 `O(log n)`。

**生僻不易理解的概念解释**

  * **向左旋转（Rotation）**：想象把数组围成一个圈，向左转动几步，然后再从某个位置剪开铺平。举个具体的例子，原本的数组是 `[1, 2, 3, 4, 5]`，如果在下标 `2` 的位置向左旋转，前两个数字 `1, 2` 会被切下来并拼接到数组的尾部，最终变成 `[3, 4, 5, 1, 2]`。
  * **时间复杂度 O(log n)**：这是算法执行效率的衡量标准。在数组问题中，如果是从头到尾逐个对比查找（线性查找），时间复杂度是 `O(n)`。而 `O(log n)` 通常意味着每次查找都要排除掉一半的可能性，这明确暗示了本题必须使用**二分查找法**。

### （2）解题思路分析

从代码逻辑的角度来剖析，由于题目强制要求 `O(log n)` 的时间复杂度，常规的从头遍历寻找不可行，必须采用**二分查找**。

普通的二分查找要求整个数组是绝对递增或递减的，但这道题的数组被“旋转”过，整体不再有序。然而，旋转数组有一个非常重要的特性：**它由两段各自从小到大排序的子数组组成，并且第一段（左段）的所有数字，都严格大于第二段（右段）的所有数字。**

代码的核心思路是利用**开区间二分查找**，并且巧妙地引入了数组的最后一个元素 `nums[-1]` 作为“标杆”，来判断当前的中间值 `x` 和目标值 `target` 分别属于哪一段数组：

1.  **判断位置区间**：
      * 如果数字大于 `nums[-1]`，说明这个数字在**第一段（左半边）**。
      * 如果数字小于等于 `nums[-1]`，说明这个数字在**第二段（右半边）**。
2.  **分情况收缩搜索区间**：
      * **情况一**：`target` 在第一段，而中间值 `x` 在第二段。此时 `target` 一定在 `x` 的左侧，需要缩小右边界（向左找）。
      * **情况二**：中间值 `x` 在第一段，而 `target` 在第二段。此时 `target` 一定在 `x` 的右侧，需要缩小左边界（向右找）。
      * **情况三**：中间值 `x` 和 `target` 在同一段（都在第一段，或都在第二段）。这时可以将这一段视为普通的有序数组。直接对比大小即可：如果 `x >= target`，说明目标在左侧或就是 `x` 本身，缩小右边界；如果 `x < target`，说明目标在右侧，缩小左边界。
3.  **循环结束**：当区间缩小到只剩下一个可能的候选位置时，退出循环并验证该位置的值是否真的是 `target`。

### （3）带有注释的代码

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        # 初始化开区间端点，搜索范围假定为 (-1, len(nums)-1)
        left, right = -1, len(nums) - 1  
        
        # 当开区间不为空（即 left 和 right 之间至少还有一个元素）时持续循环
        while left + 1 < right:  
            # 计算区间的中点位置
            mid = (left + right) // 2
            # 获取中点位置对应的具体数值
            x = nums[mid]
            
            # 判断逻辑：目标值 target 在第一段（左段），而中点值 x 在第二段（右段）
            if target > nums[-1] >= x:  
                # 说明 target 在 x 的左侧，缩小右边界至 mid
                right = mid  
                
            # 判断逻辑：中点值 x 在第一段（左段），而目标值 target 在第二段（右段）
            elif x > nums[-1] >= target:  
                # 说明 target 在 x 的右侧，缩小左边界至 mid
                left = mid  
                
            # 判断逻辑：x 和 target 同在第一段，或同在第二段。此时可以当做普通有序数组处理。
            # 如果中点值 x 大于或等于目标值 target
            elif x >= target:  
                # 说明 target 在 x 的左侧（或刚好是 x），缩小右边界至 mid
                right = mid
                
            # 判断逻辑：x 和 target 同在一段，且 x 严格小于 target
            else:
                # 说明 target 在 x 的右侧，缩小左边界至 mid
                left = mid
                
        # 循环结束后，right 就是唯一可能存在 target 的下标。最后做一次相等性校验。
        return right if nums[right] == target else -1
```

### （4）每一行代码详细的中文解释

  * `class Solution:`
    定义一个名为 `Solution` 的类，这是 LeetCode 提交代码的标准格式封装。
  * `def search(self, nums: List[int], target: int) -> int:`
    定义类中的 `search` 方法，接收一个整型列表 `nums` 和一个要查找的整数 `target`，预期返回一个代表索引的整数。
  * `left, right = -1, len(nums) - 1`
    初始化两个指针 `left` 和 `right`，采用开区间 `(left, right)` 模式。初始左边界设为 `-1`，右边界设为数组最后一个元素的下标 `len(nums) - 1`，将所有可能的答案都包裹在这个开区间内部。
  * `while left + 1 < right:`
    二分查找的循环条件。只要 `left` 和 `right` 之间还有至少一个整数距离（即开区间内还有元素），就继续执行循环。
  * `mid = (left + right) // 2`
    计算当前搜索区间的中点下标 `mid`，使用整除向下取整。
  * `x = nums[mid]`
    把中点下标 `mid` 对应的数组元素提取出来，赋值给变量 `x`，方便后续比较。
  * `if target > nums[-1] >= x:`
    进行第一次条件判断。`target > nums[-1]` 说明目标值位于由大数构成的第一段；`nums[-1] >= x` 说明当前中点值 `x` 位于由小数构成的第二段。这两者结合说明目标值在整体偏左的位置，而中点偏右。
  * `right = mid`
    承接上一步判断，既然目标值在中点的左侧，那就将右边界 `right` 移动到 `mid`，以便下一轮在左半部分继续寻找。
  * `elif x > nums[-1] >= target:`
    进行第二次条件判断。`x > nums[-1]` 说明中点值位于第一段；`nums[-1] >= target` 说明目标值位于第二段。这两者结合说明中点在偏左的大数区，而目标值在偏右的小数区。
  * `left = mid`
    承接上一步判断，既然目标值在中点的右侧，那就将左边界 `left` 移动到 `mid`，以便下一轮在右半部分继续寻找。
  * `elif x >= target:`
    如果代码执行到这里，说明 `x` 和 `target` 处于同级别的一段中（要么都在第一段，要么都在第二段）。此时直接按照普通升序数组来比较。如果中点值 `x` 大于或等于目标值 `target`。
  * `right = mid`
    由于同处于一段升序序列内，`x >= target` 意味着目标值一定在 `x` 的左侧或者正好是 `x`，因此将右边界 `right` 收缩至 `mid`。
  * `else:`
    涵盖所有不满足前面 `if` 和 `elif` 的最后一种情况，即 `x` 和 `target` 同处一段，且中点值 `x` 严格小于目标值 `target`。
  * `left = mid`
    由于同处于一段升序序列内，`x < target` 意味着目标值一定在 `x` 的右侧，因此将左边界 `left` 收缩至 `mid`。
  * `return right if nums[right] == target else -1`
    当 `while` 循环终止时，开区间内只剩下一个元素，也就是 `right` 所指向的位置。检查这个位置上的值 `nums[right]` 是否等于要找的 `target`。如果等于，返回下标 `right`；如果不等于，说明整个数组中都不存在该数字，返回 `-1`。
